RAGとは？

RAG（Retrieval-Augmented Generation）は、LLMに外部の知識を与える仕組みです。

通常のLLMは学習データにない情報（社内規定、マニュアル等）を知りません。
RAGを使うことで、自社の文書をAIが参照して回答できるようになります。

RAGの利点
① LLMを再学習させる必要がない（文書を追加するだけ）
② 出典を明示できる（「〇〇.pdf の p.5」等）
③ 情報の更新が簡単（文書を差し替えるだけ）

チャンク分割とは？

文書を小さな「チャンク（塊）」に分割してインデックス化します。
チャンクのサイズと重複（オーバーラップ）が検索精度に影響します。

設定説明推奨値
チャンクサイズ1つのチャンクの最大文字数500〜1000
オーバーラップ前後のチャンクと重複する文字数50〜100
分割方式自動 / 改行 / 段落 など通常は「自動」でOK
チャンクが小さすぎると文脈が失われる
チャンクサイズが小さすぎると、1つのチャンクに意味のある情報が収まらない。
大きすぎると検索の精度が落ちる。まずはデフォルト設定で試すのがよい。

ナレッジベースの作成手順

Step 1: ナレッジベースを作成

  1. 左メニューの「ナレッジ」をクリック
  2. 「ナレッジベースを作成」をクリック
  3. 名前をつける（例: 「社内マニュアル」）

Step 2: 文書をアップロード

  1. 「ファイルをアップロード」からPDF等を選択
  2. 複数ファイルを一括アップロード可能
  3. アップロード後、自動でチャンク分割が始まる

チャンク分割のイメージ

1つのPDFがこのように分割されます:

第1章 就業規則
勤務時間は9:00〜18:00とする。休憩時間は12:00〜13:00の1時間。 フレックスタイム制の対象者はコアタイム10:00〜15:00...
...フレックスタイム制の対象者はコアタイム10:00〜15:00とする。 有給休暇は入社6か月後に10日付与。以降、勤続年数に応じて加算...
第2章 経費精算
出張時の交通費は実費精算とする。宿泊費の上限は1泊12,000円。 事前申請が必要な場合は出張申請書を...

※ オーバーラップ部分（チャンク1の末尾とチャンク2の先頭が重複）が文脈の切れ目を防ぐ

Step 3: チャットボットにナレッジを接続

  1. Lesson 4で作ったチャットボットの編集画面を開く
  2. 「コンテキスト」セクションの「＋追加」をクリック
  3. 作成したナレッジベースを選択して接続
  4. 検索モードを選択（通常は「ベクトル検索」でOK）
検索モード特徴
ベクトル検索意味的に近い文書を検索。一般的な質問向き
全文検索キーワード完全一致で検索。固有名詞や型番の検索向き
ハイブリッド検索両方を組み合わせ。精度が高いがやや遅い

RAGの精度を上げるコツ

✅ 文書を整理してからアップロード

目次・見出しが整った文書は、チャンク分割の精度が上がる

✅ 1ナレッジベース = 1テーマ

「就業規則」「IT手順書」など分けると検索精度が向上

❌ スキャンPDFをそのまま

画像だけのPDFはテキスト抽出できない。OCR処理が必要

❌ 1ファイルに全部詰め込む

巨大な1ファイルより、適度に分割された複数ファイルの方が良い

Summary Index（v1.12〜の新機能）

AIが各チャンクの要約を自動生成し、検索時にこの要約も使って精度を向上させる機能。

通常のベクトル検索との違い
通常: 質問 → チャンクの原文と類似度比較
Summary Index: 質問 → AI生成の要約と類似度比較 → より本質的なマッチ

※ インデックス作成時にLLM APIを消費するため、コストとのバランスを考慮
