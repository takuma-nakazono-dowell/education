Difyの始め方

Difyの利用方法は2つあります。
まずはどちらで始めるか選びましょう。

クラウド版 vs セルフホスト版

項目 クラウド版（dify.ai） セルフホスト版（Docker）
セットアップ アカウント登録だけ Docker環境が必要
費用 無料枠あり / 有料プラン サーバー費用のみ
データの場所 Dify社のサーバー 自社サーバー内
管理・運用 不要（Dify社が管理） 自社で管理・更新
LLM費用 自分のAPIキーを設定 自分のAPIキーを設定
おすすめ まず試したい人 機密データを扱う場合
おすすめ: まずクラウド版で試す
操作を覚えてから必要に応じてセルフホストに移行するのが効率的。
クラウド版の無料枠で十分に学習できます。

LLM APIキーの準備

Dify自体は「プラットフォーム」であり、AIの頭脳（LLM）は別途必要です。

  • OpenAI: GPT-4o / GPT-4 を使う場合
  • Anthropic: Claude を使う場合
  • Google: Gemini を使う場合

いずれかのAPIキーを事前に取得しておきましょう。

APIキーの管理に注意
APIキーはパスワードと同じです。
他人に共有しない、GitHubにコミットしない、定期的にローテーションする。

セルフホスト版のインストール

Dockerが使える環境で以下を実行します。

# Difyリポジトリをクローン git clone https://github.com/langgenius/dify.git # dockerディレクトリに移動 cd dify/docker # 環境変数ファイルをコピー cp .env.example .env # 起動 docker compose up -d

起動後、http://localhost/install にアクセスして初期設定を行います。

Docker版の要件
Docker / Docker Compose が必要。メモリは最低4GB推奨。
社内ネットワークの制約（プロキシ等）に注意。

セットアップ手順

クラウド版の登録手順

  1. ブラウザで https://cloud.dify.ai にアクセス
  2. 「Get Started」をクリックし、メールアドレスで登録
    （GitHubアカウントやGoogleアカウントでも可）
  3. メール認証を完了してログイン
  4. ワークスペースが自動作成され、ダッシュボードが表示される

＋ 作成

ナレッジ画面でできること：

  • ナレッジベースの作成：テーマごとに文書をグルーピング
  • ファイルアップロード：PDF、Word、Excel、テキスト、HTML等に対応
  • チャンク管理：分割されたチャンクの確認・編集・削除
  • 検索テスト：実際にクエリを入力して、どのチャンクがヒットするか確認
  • インデックス方式の選択：高品質（推奨）/ エコノミー
  • メタデータ管理：ファイルごとにタグや説明を付与
インデックス方式特徴コスト
高品質Embeddingモデルでベクトル化。意味検索が可能Embedding API費用あり
エコノミーキーワードベースの全文検索のみ無料（API不要）
ナレッジ設計のコツ
1テーマ1ナレッジベースに分ける（「就業規則」「IT手順」「経理マニュアル」等）。
アプリから必要なナレッジだけを接続すると、検索精度が上がる。

3種類のツール：

種類説明例
ビルトインツール Difyに標準搭載されたツール。APIキーを設定するだけで使える Google検索、Wikipedia、天気予報、DALL-E、Stable Diffusion、計算機
カスタムツール 自社APIをOpenAPI(Swagger)スキーマで登録。任意のREST APIをツール化 社内売上API、在庫確認API、顧客検索API
ワークフローツール 作ったワークフローを他のアプリから「ツール」として呼び出せる データ整形ワークフロー、レポート生成ワークフロー

ツールの使い方例：

  • エージェント + Google検索：最新情報を調べて回答するAI（LLMの学習データにない情報も対応）
  • エージェント + カスタムAPI：「売上を調べて」→ 社内APIを叩いて回答
  • ワークフロー + DALL-E：テキストから画像を生成するステップを組み込む
  • チャットボット + 計算機：計算が必要な質問に正確に回答
カスタムツールの登録（OpenAPIスキーマ）
社内売上検索API
openapi: "3.0.0"
paths:
  /api/sales:
    get:
      summary: 売上データ取得
      parameters: ...
保存
ツール活用のポイント
① エージェントアプリでツールを有効にすると、AIが必要に応じて自動的に呼び出す
② ワークフローでは特定のノードからツールを明示的に呼び出す
③ カスタムツールはOpenAPIスキーマがあればどんなAPIでも登録可能
カスタムツールのAPI認証
外部APIにアクセスする場合、APIキーやOAuth等の認証情報が必要。
認証情報はDifyのツール設定画面で安全に管理される。
