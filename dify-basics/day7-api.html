① Webアプリとして公開

最も簡単な方法。公開ボタンを押すだけでURLが発行されます。

  • URLを社内で共有するだけですぐ使える
  • Difyの標準UIで表示される
  • アクセス制限なし（URLを知っていれば誰でも利用可能）

② サイト埋め込み

既存のWebサイトにチャットウィジェットを埋め込みます。

<!-- iframeで埋め込む場合 --> <iframe src="https://your-dify.com/chatbot/xxxx" style="width:400px; height:600px; border:none;" ></iframe> <!-- チャットバブルで埋め込む場合 --> <script> window.difyChatbotConfig = { token: 'your-app-token' } </script> <script src="https://your-dify.com/embed.min.js" defer ></script>

③ API連携

最も柔軟な方法。自社のシステムからDifyのAIアプリを呼び出します。

API連携が向いている場面
① 既存の業務システムにAI機能を追加したい
② バッチ処理で大量のデータを処理したい
③ 独自のUIで表示したい
④ 他のシステムと連携したワークフローを組みたい

APIキーの発行

  1. アプリの設定画面を開く
  2. 「APIアクセス」セクションへ移動
  3. 「APIキーを作成」をクリック
  4. 生成されたキーを安全に保管
APIキーの管理
APIキーは絶対にフロントエンド（JavaScript）に埋め込まない。
必ずバックエンド側で保持し、サーバー経由でDifyを呼び出す。

API連携の実装

基本のAPIリクエスト

DifyのAPIはRESTful形式。チャットボットの場合:

# curlでテスト（チャットメッセージ送信） curl -X POST 'https://api.dify.ai/v1/chat-messages' \ -H 'Authorization: Bearer {your-api-key}' \ -H 'Content-Type: application/json' \ -d '{ "inputs": {}, "query": "パスワードのリセット方法を教えて", "response_mode": "blocking", "user": "user-123" }'
パラメータ説明
inputsアプリに定義した変数の値（オプション）
queryユーザーのメッセージ（質問文）
response_modeblocking（一括）/ streaming（ストリーミング）
user利用者を識別するID
conversation_id会話を継続する場合に指定（オプション）

レスポンス例

{ "event": "message", "message_id": "abc-123-def", "conversation_id": "conv-456", "answer": "パスワードのリセット手順：\n1. ログイン画面の...", "metadata": { "usage": { "total_tokens": 245, "total_price": "0.000735" }, "retriever_resources": [ { "document_name": "IT手順書.pdf", "segment_position": 3, "score": 0.92 } ] } }

answer: AIの回答文
conversation_id: 次の質問で同じIDを送ると会話が継続
retriever_resources: RAGで参照した文書の情報

JavaScript（fetch）での実装例

// ⚠️ バックエンド（Node.js等）から呼ぶこと // フロントから直接呼ぶとAPIキーが漏洩する async function askDify(question, conversationId = '') { const response = await fetch('https://api.dify.ai/v1/chat-messages', { method: 'POST', headers: { 'Authorization': `Bearer ${process.env.DIFY_API_KEY}`, 'Content-Type': 'application/json' }, body: JSON.stringify({ inputs: {}, query: question, response_mode: 'blocking', user: 'user-001', ...(conversationId && { conversation_id: conversationId }) }) }); if (!response.ok) { throw new Error(`Dify API error: ${response.status}`); } return response.json(); }

C#（ASP.NET）での実装例

ストリーミングモード

ChatGPTのように文字が逐次表示される「ストリーミング」にも対応。

// response_mode を "streaming" に変更 { "response_mode": "streaming" } // Server-Sent Events (SSE) で逐次受信 // レスポンスは text/event-stream 形式 data: {"event":"message","answer":"パスワード"} data: {"event":"message","answer":"の"} data: {"event":"message","answer":"リセット手順"} data: {"event":"message_end","metadata":{...}}
