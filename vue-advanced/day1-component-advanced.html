1. emits オプション宣言 Vue 3 推奨

基礎コースで学んだ $emit は、イベントを発火するだけでした。
Vue 3 では、コンポーネントが発火するイベントを emits オプションで明示的に宣言することが推奨されています。

// ❌ 基礎コースの書き方（動くが非推奨） app.component('my-button', { methods: { handleClick() { this.$emit('clicked', 'hello') } } }) // ✅ Vue 3 推奨：emitsオプションで宣言する app.component('my-button', { emits: ['clicked'], // このコンポーネントが発火するイベント一覧 methods: { handleClick() { this.$emit('clicked', 'hello') } } })
emits宣言のメリット
・どのイベントを発火するか一目でわかる（ドキュメント代わり）
・親の同名ネイティブイベントとの衝突を防ぐ
・バリデーション関数で値の検証もできる
// emitsにバリデーション関数を付ける（高度な使い方） emits: { submit: (payload) => { // trueを返せばOK、falseで警告を出す return payload.text !== '' } }

2. v-model をコンポーネントに対応させる 新規

v-model はフォーム要素（input など）で使いますが、 自作コンポーネントにも対応させることができます。

Vue 3 の v-model は内部的にこの2つに展開されます：

<!-- この書き方は… --> <my-input v-model="keyword" /> <!-- 内部的にこう展開される --> <my-input :modelValue="keyword" @update:modelValue="keyword = $event" />

つまりコンポーネント側で modelValue を受け取り、update:modelValue を emitすれば v-model が使えるようになります：

app.component('my-input', { props: ['modelValue'], emits: ['update:modelValue'], template: ` <input :value="modelValue" @input="$emit('update:modelValue', $event.target.value)" /> ` })
なぜ直接書き換えてはいけないか
受け取った props をコンポーネント内で直接変更するのは禁止です。
代わりに「変更したい」というイベントを親に通知して、親側で更新します。 これがVueの単方向データフローのルールです。

3. provide / inject 新規

props でデータを渡す場合、コンポーネントが深くネストされていると 途中の階層が使わないのに props を受け渡し続ける問題が起きます。 これを「props のバケツリレー」と呼びます。

// バケツリレーの例（途中のコンポーネントは使わないのに受け渡す） App → Section → Panel → Detail → ★ここで必要 ↑ ↑ ↑ 使わないのにpropsを中継し続ける

provide / inject を使うと、階層をスキップして直接渡せます：

// 祖先コンポーネント（Appなど） Vue.createApp({ data() { return { appName: 'ORSYSシステム', accentColor: '#c62828' } }, provide() { // provideは関数にするとthisが使える return { appName: this.appName, accentColor: this.accentColor } } }) // 深い階層の子孫コンポーネント { inject: ['appName', 'accentColor'], template: `<div :style="{ color: accentColor }">{{ appName }}</div>` }
provide / inject の使いどころ
・アプリ全体で使う設定値（テーマ・ロケール・API URLなど）
・コンポーネントライブラリで親→子孫へ状態を共有するとき
・頻繁に更新されないデータに向いている

App → Section → Card という3階層構造です。

// 最下層の Card コンポーネント const InfoCard = { inject: ['systemName', 'headerColor'], template: ` <div style="border-radius:8px;overflow:hidden;box-shadow:0 2px 8px rgba(0,0,0,0.12);"> <div :style="{ background: headerColor, color: 'white', padding: '8px 14px' }"> {{ systemName }} </div> <div style="padding:12px;background:white;"> このカードは inject でシステム名と色を取得しています。<br> Section には props を渡していません。 </div> </div> ` } // 中間の Section（provideもinjectも不要） const AppSection = { components: { InfoCard }, template: ` <div> <p style="font-size:0.88em;color:#666;">← Section（中継なし）</p> <info-card /> </div> ` } // 祖先の App で provide Vue.createApp({ components: { AppSection }, provide() { return { systemName: 'ORSYSシステム', headerColor: '#c62828' } }, template: `<app-section />` }).mount('#demo3')

number-input コンポーネントは modelValue を props で受け取り、
