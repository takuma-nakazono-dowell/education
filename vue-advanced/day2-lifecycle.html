ライフサイクルとは？

Vueのコンポーネントには「生まれてから消えるまで」の一定の流れがあります。 この流れの各タイミングで処理を差し込める仕組みがライフサイクルフックです。

beforeCreateインスタンス生成直後（data未初期化）
↓
createddata / computed 利用可能、DOM未生成
↓
beforeMountDOM生成直前
↓
mounted★ DOM利用可能。最もよく使う
↓ データ変更があるたびに
beforeUpdate再レンダリング前
↓
updated★ 再レンダリング後のDOM利用可能
↓ コンポーネント削除時
beforeUnmount削除直前
↓
unmounted★ 削除完了。クリーンアップに使う

よく使うフック

① created

data が使えるが DOM はまだない。非同期処理の初期データ取得によく使う。

created() { // この時点でthis.dataが使える this.userId = this.getQueryParam('id') }

② mounted ★

DOM が確定した直後。DOM を直接操作する処理やAPIからデータ取得に使う。

mounted() { // DOM要素を直接参照できる（$el, $refs） this.$refs.myInput.focus() // Day 3 で詳しく学ぶ：APIデータ取得 this.fetchData() }

③ updated

データが変わって再レンダリングされた後。更新後の DOM を参照したいときに使う。

updated() { // リスト更新後に末尾へスクロールする例 const list = this.$refs.chatList list.scrollTop = list.scrollHeight }

④ unmounted ★

コンポーネントが画面から消えたとき。タイマーやイベントリスナーの後始末に必ず使う。

mounted() { // タイマーを起動して保存しておく this.timer = setInterval(() => { this.time++ }, 1000) }, unmounted() { // コンポーネント削除時に必ずタイマーを止める // ★ これを忘れるとメモリリークの原因になる clearInterval(this.timer) }
クリーンアップを忘れずに
setInterval・addEventListener・WebSocket などを mounted で開始したら、 unmounted で必ず止めてください。 コンポーネントが消えても動き続け、メモリリークの原因になります。

$refs でDOM要素を直接参照

テンプレートに ref="名前" を付けると、this.$refs.名前 で DOM 要素を取得できます。

<!-- テンプレート --> <input ref="searchBox" /> // JS側 mounted() { this.$refs.searchBox.focus() // 画面表示時に自動フォーカス }

練習: 経過時間を表示するコンポーネント

マウントされてからの経過秒数を表示するコンポーネントを作ってください。
・mounted でタイマーを開始する
・unmounted でタイマーを停止する（メモリリーク防止）
・「表示 / 非表示」で切り替えたとき、非表示にするとリセットされること
ヒント
data() に elapsed: 0 と timer: null を用意します。
mounted で setInterval(() => this.elapsed++, 1000) を実行して結果を this.timer に保存。
unmounted で clearInterval(this.timer) を呼びます。
答えを見る
app.component('elapsed-timer', { data() { return { elapsed: 0, timer: null } }, mounted() { this.timer = setInterval(() => { this.elapsed++ }, 1000) }, unmounted() { clearInterval(this.timer) }, template: ` <p style="font-size:1.2em;"> マウントから <strong>{{ elapsed }}</strong> 秒経過 </p> ` })
