SQL 演習問題

手術室システムのデータを使った実践的なSQL演習

この教材について

病院の手術室管理システムのデータを使って、実務レベルのSQLを練習します。

各問題はブラウザ上で実行できます。AIは使わずに自力で考えてください。

わからない場合は SQL の教科書・リファレンスを参照してOKです。

使用するテーブル

operation_table（症例）
orsys_idINTEGERPK
patient_idTEXT
patient_nameTEXT
age_yearINTEGER
age_monthINTEGER
sex_nameTEXT
birthdayTEXT
heightTEXT
weightTEXT
ope_extra_idINTEGER
section_idINTEGER
ope_extra_id: 1=通常 2=緊急 3=枠外
operation_schedule_table（スケジュール）
orsys_idINTEGERPK/FK
ope_dateTEXT
sche_ope_start_timeTEXT
sche_ope_end_timeTEXT
ope_room_idINTEGER
時刻は DATETIME 形式（'YYYY-MM-DD HH:MM:SS'）
operation_name_table（術式）
orsys_idINTEGERFK
operation_name_idINTEGER
operation_nameTEXT
idx_dspINTEGER
1症例に複数術式あり
operation_position_table（体位）
orsys_idINTEGERFK
operation_position_idINTEGER
operation_position_nameTEXT
idx_dspINTEGER
section_master（診療科マスタ）
section_idINTEGERPK
section_nameTEXT
idx_dspINTEGER
idx_dsp: 表示順（昇順ソート基準）
ope_room_master（手術室マスタ）
ope_room_idINTEGERPK
ope_room_nameTEXT
idx_dspINTEGER
テーブル関連 operation_table.orsys_id = operation_schedule_table.orsys_id = operation_name_table.orsys_id = operation_position_table.orsys_id
operation_table.section_id = section_master.section_id
operation_schedule_table.ope_room_id = ope_room_master.ope_room_id

カリキュラム

Phase
1

絞り込み・基本検索

WHERE / JOIN / LIKE / 日付・時刻フィルタ

  • 2020年4月の手術をすべて抽出
  • 2020年1年間の緊急手術をすべて抽出
  • 16歳未満の手術をすべて抽出
  • 「腹腔鏡」の手術（1症例1レコード）
  • 「腹臥位」の手術をすべて抽出
  • 心臓血管外科・緊急/枠外・70代
  • 午前中（AM11:59まで）の手術
  • 深夜（PM10時以降〜AM5時）の緊急手術
  • 手術時間をN時間M分で表示
Phase
2

集計・グループ化

GROUP BY / COUNT / AVG / 多軸集計

  • 手術日毎の件数を集計
  • 年毎・年月毎の件数を集計
  • 年月×診療科、年月×術式、年月×診療科×術式
  • 平均手術時間の算出
  • 手術室毎の件数・平均時間
  • TOP10術式（その他を含む）
Phase
3

応用・分析

サブクエリ / NOT EXISTS / CASE / 比率計算

  • 手術が行われていない診療科を抽出
  • BMIを計算（身長・体重NULLを除外）
  • 午前中で件数が最多の診療科
  • 深夜緊急手術で件数が最多の診療科
  • 日跨ぎの手術件数
  • 利用頻度が最も高い手術室
  • 女性比率が高い診療科（比率付き）
