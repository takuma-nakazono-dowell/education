条件とリスト

Vueのディレクティブを使うと、条件に応じた表示の切り替えや、 配列データをリスト表示する処理をHTMLに直接書けます。

v-if / v-else-if / v-else

条件によって要素を表示・非表示にします。DOMから要素自体を削除します。

<div v-if="score >= 90">A評価</div> <div v-else-if="score >= 70">B評価</div> <div v-else-if="score >= 50">C評価</div> <div v-else>D評価（50未満）</div> // data: { score: 85 } → 「B評価」が表示される
v-if の特徴
条件が false の場合、要素はDOMから完全に削除されます。 要素を初期化しながら切り替えたい場合に適しています。

v-show との違い

ディレクティブ false のとき 適した用途
v-if DOMから削除（HTML上に存在しない） 初期表示で非表示にしたい場合
v-show display:none で隠す（DOMには残る） 頻繁に表示/非表示を切り替える場合
<!-- v-show は CSS で display を切り替えるだけ --> <div v-show="isVisible"> このコンテンツはDOMに残ったまま表示/非表示が切り替わる </div>

v-for でリスト表示

配列の各要素を繰り返しレンダリングします。

<!-- 配列をループ --> <ul> <li v-for="item in items" :key="item.id"> {{ item.name }} - {{ item.price }}円 </li> </ul> // data: { items: [{ id:1, name:'りんご', price:150 }, ...] } <!-- インデックス番号も取得できる --> <li v-for="(item, index) in items" :key="item.id"> {{ index + 1 }}. {{ item.name }} </li>

:key 属性の重要性

:key は必須！
v-for を使うときは必ず :key を指定します。 VueがリストのどのDOM要素を更新すべきか識別するために使います。 keyには各アイテムで一意の値（IDなど）を使います。 配列のインデックス（index）は順序が変わると問題が起きるため非推奨です。

v-for でオブジェクトを繰り返す

<!-- オブジェクトのプロパティをループ --> <div v-for="(value, key) in userInfo" :key="key"> {{ key }}: {{ value }} </div> // data: { userInfo: { name:'田中', age:25, city:'東京' } } // → name: 田中 / age: 25 / city: 東京

v-if と v-for の組み合わせ

<!-- フィルタリング表示（v-for の中で v-if を使う）--> <li v-for="item in items" :key="item.id" v-if="item.inStock"> {{ item.name }}（在庫あり） </li> // ※ v-if と v-for を同じ要素に使うとき、 // 実務では computed でフィルタ済み配列を作る方が推奨

問題 1: v-if と v-show の使い分け

「ドロップダウンメニュー」のように頻繁に表示/非表示を切り替える場合、 v-if と v-show どちらが適していますか？その理由も答えてください。
答えを見る
v-show が適しています。

理由: v-if は条件が変わるたびにDOMの要素を破棄・再生成するため、 頻繁な切り替えではコストがかかります。
v-show は最初にDOMを生成し、その後は display プロパティで 表示/非表示を切り替えるだけなので、高速に動作します。

目安: 頻繁に切り替える → v-show / 初期非表示・条件が滅多に変わらない → v-if

問題 2: :key の重要性

以下のコードで :key に index を使うと どのような問題が起きる可能性がありますか？
<li v-for="(item, index) in items" :key="index"> {{ item.name }} </li>
答えを見る
リストの先頭に新しいアイテムを挿入した場合、 すべての要素のインデックスがズレてしまいます。
Vueはkeyが変わった要素を「別の要素」として扱うため、 不必要な再レンダリングが起きたり、 チェックボックスの状態がズレるなどのバグが発生します。

対策: 各アイテム固有の id など、 変わらない一意の値をkeyに使います。

問題 3: v-for の構文

以下のデータから、名前と職業を表示するv-forを完成させてください。
// data users: [ { id: 1, name: '田中', job: 'エンジニア' }, { id: 2, name: '山田', job: 'デザイナー' }, { id: 3, name: '佐藤', job: 'ディレクター' } ] // 穴埋め <div v-for="___ in ___" :key="___"> {{ ___.name }}: {{ ___.job }} </div>
答えを見る
<div v-for="user in users" :key="user.id"> {{ user.name }}: {{ user.job }} </div>
v-for="user in users" で配列 users の各要素を user として取得。 :key="user.id" で一意のIDをkeyに指定します。
