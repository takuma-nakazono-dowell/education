コンポーネントとは？

コンポーネントとは、UIを再利用可能な部品に分割する仕組みです。 同じようなHTMLを何度も書く代わりに、コンポーネントとして定義して使い回せます。

親コンポーネント（Vueアプリ）
user-card
name="田中"
user-card
name="山田"
user-card
name="佐藤"
同じコンポーネントを、異なるデータ（props）で使い回す
コンポーネントのメリット
・同じUIを何度も書かなくていい（再利用性）
・コードをパーツに分けて管理しやすくなる（保守性）
・大きなプロジェクトでも構造を整理できる

app.component() でグローバルコンポーネントを定義

const app = Vue.createApp({ ... }) // コンポーネントの定義 app.component('user-card', { // props: 親から受け取るデータを宣言する props: ['name', 'age'], // template: このコンポーネントのHTML template: ` <div class="card"> <h3>{{ name }}</h3> <p>{{ age }}歳</p> </div> ` }) app.mount('#app')

コンポーネントを使う（HTML側）

<div id="app"> <!-- コンポーネントタグを使う --> <user-card name="田中太郎" age="25"></user-card> <user-card name="山田花子" age="22"></user-card> <!-- 動的なデータを渡す場合は v-bind を使う --> <user-card v-for="user in users" :key="user.id" :name="user.name" :age="user.age" ></user-card> </div>

props で親から子へデータを渡す

// 型定義付きのprops（推奨） props: { name: { type: String, // 型チェック required: true // 必須かどうか }, age: { type: Number, default: 0 // デフォルト値 }, isAdmin: { type: Boolean, default: false } }
props の流れ（一方向）
親 → 子 への一方向のデータフローです。 子コンポーネントは受け取ったpropsを直接変更してはいけません。 変更が必要な場合は $emit でイベントを発火して親に通知します。

$emit でカスタムイベントを発火（子→親）

// 子コンポーネント app.component('like-button', { props: ['label'], data() { return { count: 0 } }, methods: { handleClick() { this.count++ // 親に 'liked' イベントを発火（カウントを渡す） this.$emit('liked', this.count) } }, template: ` <button @click="handleClick"> ♡ {{ label }} ({{ count }}) </button> ` }) // 親側でイベントを受け取る <like-button label="いいね" @liked="onLiked" ></like-button>
命名規則: ケバブケース
コンポーネント名とカスタムイベント名はHTMLでは kebab-case（ハイフン区切り）で書きます。
例: user-card、@liked、@item-selected

問題 1: props の受け渡し

以下のコンポーネント定義で、HTMLから正しく props を渡している書き方はどれですか？
app.component('product-card', { props: ['title', 'price'], template: `<div>{{ title }}: {{ price }}円</div>` }) // 使い方 A（静的文字列） <product-card title="りんご" price="150"></product-card> // 使い方 B（動的バインディング） <product-card :title="product.name" :price="product.price"></product-card> // 使い方 C（{{ }} をHTML属性に使う） <product-card title="{{ product.name }}" price="{{ product.price }}"></product-card>
答えを見る
A と B の両方が正しい（Cは間違い）

A: 固定値を渡す場合は属性にそのまま書けます。ただし数値も文字列になります。
B: :title でデータを動的に渡します。数値は数値として渡せます。
C: HTML属性内で {{ }} は使えません。必ず :（v-bind）を使います。

問題 2: $emit の流れ

子コンポーネントが this.$emit('delete', item.id) を実行したとき、 親側でこのイベントを受け取るには何と書きますか？
答えを見る
<!-- 親テンプレート --> <my-component @delete="handleDelete"></my-component> // 親の methods handleDelete(itemId) { // itemId に子から渡された item.id が入る console.log('削除:', itemId) }
イベント名 'delete' を @delete でリッスンし、 メソッドで受け取ります。 $emit の第2引数以降がメソッドの引数として渡されます。

問題 3: props の型定義の使い分け

以下のprops定義の違いを説明してください。
// 書き方 A（配列形式） props: ['name', 'age'] // 書き方 B（オブジェクト形式） props: { name: { type: String, required: true }, age: { type: Number, default: 0 } }
答えを見る
書き方 A（配列形式）: シンプルだが型チェックなし。 プロトタイプや小規模では問題ないが、間違った型のデータを渡しても警告が出ない。

書き方 B（オブジェクト形式）: 型・必須/任意・デフォルト値を指定できる。 間違った型のpropsを渡したとき開発者コンソールに警告を出してくれる。

推奨は B。チームで開発する場合や、中規模以上のプロジェクトでは型定義付きを使いましょう。
