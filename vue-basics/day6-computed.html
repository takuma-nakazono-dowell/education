computed（算出プロパティ）

computed（コンピューテッド）は、 他のデータをもとに自動計算される読み取り専用のプロパティです。 依存するデータが変わると自動的に再計算されます。

Vue.createApp({ data() { return { firstName: '田中', lastName: '太郎', items: [ { name: 'りんご', price: 150 }, { name: 'バナナ', price: 80 } ] } }, computed: { // 姓と名を結合するcomputedプロパティ fullName() { return this.lastName + ' ' + this.firstName }, // 合計金額を計算 totalPrice() { return this.items.reduce((sum, item) => sum + item.price, 0) } } })
<!-- テンプレートで普通のデータと同じように使える --> <p>フルネーム: {{ fullName }}</p> <p>合計: {{ totalPrice }}円</p>

computed と methods の違い（キャッシュ）

computed methods
キャッシュ 依存データが変わるまでキャッシュされる 呼ばれるたびに毎回実行される
呼び方 {{ fullName }}（括弧なし） {{ getFullName() }}（括弧あり）
適した用途 値の計算・変換・フィルタリング ボタンクリック・副作用のある処理
キャッシュとは？
computedは依存するデータが変わらない限り、計算を再実行しません。 重い計算でも同じ結果が必要なときはキャッシュから返すので高速です。 methodsは呼ばれるたびに毎回計算します。
// computedでリストをフィルタリング（実用的な例） computed: { filteredItems() { return this.items.filter(item => item.name.includes(this.searchText) ) } } <!-- テンプレートではシンプルに使える --> <li v-for="item in filteredItems" :key="item.id"> {{ item.name }} </li>

watch（ウォッチャー）

watch は、特定のデータが変化したときに処理を実行する仕組みです。 computedと違い、副作用（API呼び出し・ローカルストレージ保存など）を伴う処理に使います。

watch: { // searchText が変わるたびに実行される searchText(newValue, oldValue) { console.log(`変更: ${oldValue} → ${newValue}`) this.fetchResults(newValue) // API呼び出しなど }, // オブジェクトの深い変化を監視（deep: true） userProfile: { handler(newVal) { localStorage.setItem('profile', JSON.stringify(newVal)) }, deep: true } }

computed vs watch の使い分け

状況 使うべきもの
他のデータから値を計算・加工したい computed
データ変化に応じてAPI呼び出しをしたい watch
リストをフィルタリングしたい computed
データをローカルストレージに保存したい watch
入力値のバリデーション結果を表示 computed

問題 1: computed の特徴

以下のcomputed と methods の呼び方の違いを説明してください。
computed: { fullName() { ... } } methods: { getFullName() { ... } } // テンプレートでの使い方 {{ fullName }} // A {{ getFullName() }} // B
答えを見る
A（computed）: 括弧なしで書きます。データのように扱います。 依存するデータが変わらない限りキャッシュされ、毎回計算しません。

B（methods）: 括弧ありで呼び出します。 テンプレートが再レンダリングされるたびに毎回実行されます。

値を計算して表示するだけならcomputedが効率的です。

問題 2: watch vs computed の選択

以下のそれぞれの処理に computed と watch のどちらが適していますか？
  1. firstName と lastName を結合してフルネームを表示する
  2. ユーザーが入力したテキストをローカルストレージに自動保存する
  3. 配列を価格順に並び替えたリストを表示する
  4. 検索ワードが変わったらAPIを呼び出して結果を取得する
答えを見る
  1. computed（値の計算・変換）
  2. watch（副作用: ローカルストレージへの書き込み）
  3. computed（データの変換・フィルタリング）
  4. watch（副作用: API呼び出し）
シンプルな判断基準:
「値を計算して返すだけ」→ computed
「データ変化を検知して何か処理をしたい（副作用あり）」→ watch

問題 3: 穴埋め — computedでフィルタリング

searchText に含まれるキーワードで items 配列を フィルタリングするcomputedを完成させてください。
computed: { filteredItems() { return this.items.___(item => item.name.___(this.searchText) ) } }
答えを見る
computed: { filteredItems() { return this.items.filter(item => item.name.includes(this.searchText) ) } }
Array.filter() で条件に合う要素だけ取り出し、 String.includes() でテキストが含まれるか確認します。 大文字/小文字を区別しない場合は両方に .toLowerCase() をかけます。
