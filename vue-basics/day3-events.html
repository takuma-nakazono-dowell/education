イベント処理

ユーザーのクリックやキー入力などの操作に反応する処理をイベント処理と呼びます。 Vue.jsでは v-on ディレクティブ（または @ の短縮形）を使います。

v-on:click → @click

<!-- v-on を使った書き方 --> <button v-on:click="count++">増やす</button> <!-- @ を使った短縮記法（こちらをよく使う）--> <button @click="count++">増やす</button> <!-- メソッドを呼び出す場合 --> <button @click="increment">増やす</button>

methods オプションでメソッドを定義

複雑な処理は methods にまとめて書きます。

Vue.createApp({ data() { return { count: 0 } }, methods: { // this でデータにアクセスできる increment() { this.count++ }, decrement() { if (this.count > 0) this.count-- }, reset() { this.count = 0 } } }).mount('#app')
this とは？
methods の中で this を使うと、 data() で定義したデータや他のメソッドにアクセスできます。 this.count で count データを参照・変更できます。

イベントオブジェクト $event

<!-- $event でイベントオブジェクトを受け取る --> <input @input="onInput($event)"> methods: { onInput(event) { // event.target.value で入力値を取得 console.log(event.target.value) } } <!-- 引数なしの場合は自動で渡される --> <input @input="onInput"> // ↑ この場合 onInput(event) の event に自動で入る

@input でリアルタイム入力検知

<input @input="text = $event.target.value"> <p>文字数: {{ text.length }}</p> <!-- v-model を使えばもっとシンプルに --> <input v-model="text"> <p>文字数: {{ text.length }}</p>

@submit.prevent でフォーム送信防止

<!-- .prevent は event.preventDefault() と同じ --> <form @submit.prevent="handleSubmit"> <input v-model="name"> <button type="submit">送信</button> </form>

キー修飾子

  • @keyup.enter — Enterキーを離したとき
  • @keyup.esc — Escキーを離したとき
  • @keydown.tab — Tabキーを押したとき
  • @click.stop — クリックイベントの伝播を停止
  • @click.prevent — デフォルト動作を防止
  • @click.once — 1回だけ発火する
<!-- Enterキーで検索を実行 --> <input v-model="searchText" @keyup.enter="search" placeholder="Enterで検索">

問題 1: @click の書き方

ボタンをクリックしたとき sayHello メソッドを呼ぶコードで、 正しい短縮記法はどれですか？
// A <button onclick="sayHello"> // B <button v-on:click="sayHello"> // C <button @click="sayHello"> // D BとCは同じ意味で両方正しい
答えを見る
D: B と C の両方が正しい
v-on:click と @click は同じ意味です。 Aの onclick（VanillaJS）はVueアプリ内でも使えますが、 Vueのリアクティブ機能と連携しないため使いません。

問題 2: methods の this

以下のコードで this.message はどのデータを参照しますか？
Vue.createApp({ data() { return { message: 'Hello!', count: 0 } }, methods: { showMessage() { alert(this.message) // ← これは？ } } })
答えを見る
this.message は data() で定義した "Hello!" を参照します。

methods の中では this がVueインスタンスを指すため、 this.データ名 でデータに、 this.メソッド名() で他のメソッドにアクセスできます。

問題 3: 修飾子の使い方

フォームをサブミットしたとき、ページのリロードを防ぎながら onSubmit メソッドを呼ぶ正しい書き方はどれですか？
// A <form @submit="onSubmit"> // B <form @submit.prevent="onSubmit"> // C <form @submit.stop="onSubmit"> // D <form v-on:submit.prevent="onSubmit">
答えを見る
B と D の両方が正しい（短縮形の違いだけ）
.prevent 修飾子は event.preventDefault() を自動で呼んでくれます。 フォームのデフォルト動作（ページリロード）を防ぎながら、 Vueのメソッドで処理できます。

Aは修飾子なしなのでページがリロードされてしまいます。 Cの .stop はイベントの伝播を止めるものでフォーム送信防止には使いません。

問題 4: デモ改造チャレンジ

デモ1のカウンターに「×2」ボタンを追加するには、どのようなコードを書けばよいですか？
答えを見る
HTMLに追加:
<button @click="double">×2</button>
methods に追加:
double() { this.count *= 2 }
this.count *= 2 は this.count = this.count * 2 と同じ意味です。 methodsに新しいメソッドを追加し、ボタンの@clickで呼び出すだけです。
