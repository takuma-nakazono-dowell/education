データバインディングとは？

データバインディングとは、JavaScriptのデータとHTMLの表示を連動させる仕組みです。 Vue.jsでは、データが変わると自動的にHTMLが更新されます。これをリアクティブと呼びます。

リアクティブとは？
データが変化すると、それを使っているすべての場所が自動的に更新される性質のことです。 手動でDOMを操作する必要がありません。

v-bind でHTML属性にデータをバインド

{{ }} はテキストコンテンツに使いますが、 HTML属性にデータを埋め込むには v-bind を使います。

<!-- v-bind を使う書き方 --> <a v-bind:href="url">リンク</a> <img v-bind:src="imageUrl"> <!-- : を使った短縮記法（こちらをよく使う）--> <a :href="url">リンク</a> <img :src="imageUrl"> <button :disabled="isLoading">送信</button>
v-bind の短縮記法
v-bind:href は :href と書けます。 コロン1文字が v-bind の省略形です。実務ではほぼ必ず短縮形を使います。

v-bind:class でクラスを動的切り替え

<!-- isActive が true なら 'active' クラスが付く --> <div :class="{ active: isActive }"> コンテンツ </div> <!-- 複数のクラスを条件で切り替え --> <div :class="{ active: isActive, error: hasError }"> コンテンツ </div> <!-- 変数でクラス名を直接指定 --> <div :class="buttonClass">ボタン</div>

v-bind:style でスタイルを動的変更

<!-- スタイルをオブジェクトで指定 --> <div :style="{ color: textColor, fontSize: fontSize + 'px' }"> テキスト </div> // data() での定義 data() { return { textColor: '#42b883', fontSize: 20 } }

v-model で双方向バインディング

v-model は、フォーム要素（input, textarea, select）と データを双方向に連携させます。 ユーザーが入力すると自動的にデータが更新され、画面も更新されます。

<input v-model="name"> <p>こんにちは、{{ name }}さん！</p> <!-- チェックボックスとの連携 --> <input type="checkbox" v-model="agreed"> <p>同意: {{ agreed }}</p> <!-- セレクトボックスとの連携 --> <select v-model="selected"> <option>赤</option> <option>青</option> </select>
v-bind と v-model の違い
v-bind（:）は一方向（データ → 画面）
v-model は双方向（データ ⇔ 画面）
フォームには v-model、表示だけには v-bind/:を使います。

動くデモ & 練習問題

デモ 1: v-model — リアルタイム双方向バインディング

テキストを入力すると、下の表示がリアルタイムに変わります。

こんにちは、{{ name || '（名前未入力）' }} さん！
文字数: {{ name.length }} 文字

デモ 2: v-bind:class — クラスを動的に切り替え

ボタンをクリックすると、クラスが切り替わりスタイルが変化します。

状態: {{ isActive ? '✓ アクティブ' : '✗ 非アクティブ' }}

isActive の値: {{ isActive }}

問題 1: v-bind の使い方を選ぼう

画像のsrc属性に変数 imgUrl を使うとき、 正しい書き方はどれですか？
// A <img src="{{ imgUrl }}"> // B <img v-bind:src="imgUrl"> // C <img :src="imgUrl"> // D（BとCは同じ意味） BとCの両方が正しい
答えを見る
D: B と C の両方が正しい
v-bind:src="imgUrl" と :src="imgUrl" は同じ意味です。
Aの src="{{ imgUrl }}" は間違いです。 HTML属性の中では {{ }} は使えません。 属性には必ず v-bind: または : を使います。

問題 2: v-model の穴埋め

テキストエリアと段落を双方向バインディングするコードを完成させましょう。
<textarea ___="message"></textarea> <p>入力内容: {{ ___ }}</p> data() { return { message: '' } }
答えを見る
<textarea v-model="message"></textarea>
<p>入力内容: {{ message }}</p>

テキストエリアに v-model="message" を指定します。 テンプレート側は通常の {{ message }} でOKです。

問題 3: v-bind と v-model の使い分け

以下のそれぞれのケースで v-bind と v-model どちらを使うべきですか？
  1. ユーザーが入力するテキストフィールド
  2. 画像のsrc属性にURLを設定する
  3. プルダウンメニューの選択値を取得する
  4. ボタンのdisabled属性を条件で切り替える
答えを見る
  1. v-model（ユーザー入力 → 双方向）
  2. v-bind（:src）（属性設定 → 一方向）
  3. v-model（ユーザー選択 → 双方向）
  4. v-bind（:disabled）（属性設定 → 一方向）
ポイント: ユーザーが操作するフォーム要素 → v-model、HTMLの属性に値を設定 → v-bind（:）

問題 4: 応用 — カラーピッカーを作ろう

以下のコードを見てください。何が表示されますか？また、color の値を変えると何が変わりますか？
<div :style="{ background: color, padding: '20px', borderRadius: '8px' }"> 選択中の色: {{ color }} </div> <input type="color" v-model="color"> data() { return { color: '#42b883' } }
答えを見る
緑色の背景を持つdivが表示される。 カラーピッカー（input type="color"）で色を選ぶと、 v-model によってcolorが更新され、 :style の背景色がリアルタイムに変わります。

これが「データバインディング + リアクティブ」の力です。 DOMを一切手動操作していません！
