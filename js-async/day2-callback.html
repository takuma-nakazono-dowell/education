コールバック関数とは？

「後で呼び出してください」と渡す関数のことです。
非同期処理が完了したときに実行してほしい処理を、関数として渡します。

// コールバックの基本形 function doSomething(callback) { // 処理が終わったらcallbackを呼ぶ setTimeout(function() { callback("処理完了"); }, 1000); } doSomething(function(result) { console.log(result); // 1秒後に "処理完了" });
ポイント: 関数を「引数として渡す」
JavaScriptでは関数を変数と同じように渡せます。
コールバックは「処理が終わったら呼ぶ関数」として使われます。

実際の例: setTimeout のコールバック

// setTimeout の第1引数がコールバック setTimeout(function() { console.log("2秒後に実行！"); }, 2000); // ↑ これがコールバック関数

コールバック地獄

非同期処理を順番に実行しようとすると、コールバックの中にコールバックが…と深くなっていきます。

// ① ユーザーを取得 getUser(userId, function(user) {
// ② そのユーザーの注文を取得 getOrders(user.id, function(orders) {
// ③ 最新注文の詳細を取得 getOrderDetail(orders[0].id, function(detail) {
// ④ 商品情報を取得 getProduct(detail.productId, function(product) {
console.log(product.name);
});
});
});
});
コールバック地獄の問題点
  • ネストが深くなり読みにくい（ピラミッド型）
  • エラーハンドリングを各段階で書く必要がある
  • 処理の流れを追うのが困難
  • 修正・テストが難しい
この問題を解決するために Promise が生まれました（Day 3）。

エラーハンドリングの難しさ

// 各段階でエラー処理が必要... getUser(id, function(err, user) { if (err) { handleError(err); return; } getOrders(user.id, function(err, orders) { if (err) { handleError(err); return; } // ...さらにネストが続く }); });
