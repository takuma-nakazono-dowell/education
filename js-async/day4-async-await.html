async/await とは？

Promise をさらに同期処理のように書ける構文です。
中身は Promise と同じですが、コードが格段に読みやすくなります。

async 関数

// async をつけると Promise を返す関数になる async function fetchUser() { return { id: 1, name: "田中" }; // return は自動で Promise.resolve(値) になる } // 呼び出し側 fetchUser().then(user => console.log(user));

await — Promise の結果を待つ

async function main() { // await = この行でPromiseが解決するまで待つ const user = await fetchUser(); console.log(user.name); const orders = await fetchOrders(user.id); console.log(orders); } // await は async関数の中でしか使えない！

try/catch でエラーハンドリング

async function loadData() { try { const data = await fetchFromAPI(); console.log("成功:", data); } catch (err) { console.log("エラー:", err.message); } finally { console.log("常に実行"); } }

Promise チェーン vs async/await の比較

Promise チェーン

getUser(id) .then(u => getOrders(u.id)) .then(o => getDetail(o[0].id)) .then(d => console.log(d)) .catch(err => console.log(err));

async/await

try { const u = await getUser(id); const o = await getOrders(u.id); const d = await getDetail(o[0].id); console.log(d); } catch (err) { console.log(err); }
async/await が好まれる理由
変数に名前をつけられる・条件分岐が自然に書ける・デバッグしやすい。
現代のJavaScript開発では async/await が主流です。

注意: await は並列に使わない

// ❌ 遅い：順番に実行（合計2.5秒） const a = await api1(); // 1秒待つ const b = await api2(); // 1.5秒待つ // ✅ 速い：並列実行（最大1.5秒） const [a, b] = await Promise.all([api1(), api2()]);
