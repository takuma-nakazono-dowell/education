① UIをブロックしない

非同期処理中でも画面は応答し続けなければなりません。
ローディング中であることをユーザーに伝えましょう。

❌ 悪い例：処理中にボタンをクリックできてしまう
async function loadData() { // ボタンを無効化しない const data = await fetchFromAPI(); showData(data); }
✅ 良い例：ローディング状態を管理する
async function loadData() { setLoading(true); // ボタン無効化・スピナー表示 try { const data = await fetchFromAPI(); showData(data); } catch (err) { showError(err.message); } finally { setLoading(false); // 必ず解除 } }

デモ: ローディング状態の管理

「悪い例」は処理中に何度でもクリックできます。「良い例」はボタンが無効化されます。

② エラーを必ずハンドリングする

catch なしで await は絶対にやってはいけない
業務システムでは「サーバーが落ちている」「タイムアウト」「認証切れ」など様々なエラーが起きます。 ハンドリングしないとユーザーに何も表示されず、バグに気づけません。
// ❌ エラーハンドリングなし（NG） const data = await fetchOrders(); // エラー時に画面が固まる // ✅ エラーハンドリングあり（推奨） try { const data = await fetchOrders(); showOrders(data); } catch (err) { showErrorMessage('注文データの取得に失敗しました'); console.error(err); // ログに詳細を残す }

③ 並列 vs 直列 — 適切な使い分け

直列（順番に実行）

前の結果を次で使う場合に使う。

// ユーザー取得 → そのIDで注文取得 const user = await getUser(id); const orders = await getOrders(user.id); // user.id が必要なので直列OK

並列（同時に実行）

お互いに依存しない場合に使う。

// 独立したデータを同時取得 const [users, products] = await Promise.all([ getUsers(), getProducts() ]); // 互いに無関係 → 並列が速い

④ ORSYSのような業務システムでの考慮点

  • 🔐
    認証トークンの付与 APIリクエストには必ず認証ヘッダーを付ける。セッション切れ時は再ログイン誘導。
  • ⏱️
    タイムアウト設定 業務APの場合は処理が重いこともある。適切なタイムアウトを設定し、「応答なし」状態を防ぐ。
  • 🔁
    二重送信防止 「登録」ボタンを連打されると同じデータが複数作成される。送信中はボタンを無効化する。
  • 📋
    楽観的 vs 悲観的UI更新 先に画面を更新してからAPI送信（楽観的）か、確認後に更新（悲観的）か。業務データは悲観的が安全。
  • 📊
    大量データの取得に注意 一度に全件取得せずページネーション（1ページ50件等）を使う。フロントでのフィルタリングは件数が多いと遅い。
  • 🔄
    Vueでの非同期処理 mounted() や created() の中で async/await を使う。v-if で「データが来るまで表示しない」制御も忘れずに。

まとめ: 非同期の鉄則

  • ① ローディング状態を管理する（開始時にON、finallyでOFF）
  • ② 必ず try/catch でエラーを処理する
  • ③ 独立した処理は Promise.all で並列化（依存関係があるなら直列）
  • ④ 送信中のボタンを無効化して二重送信を防ぐ
  • ⑤ 大量データは一括取得しない（ページネーション）
