Promise とは？

「非同期処理の結果を将来届けることを約束するオブジェクト」です。
コールバック地獄を解決し、非同期処理をチェーンで書けるようにしました。

3つの状態

pending
待機中
処理中
fulfilled
成功
resolve()が呼ばれた
rejected
失敗
reject()が呼ばれた

Promiseの作り方

const promise = new Promise((resolve, reject) => { setTimeout(() => { const success = true; if (success) { resolve("取得成功"); // fulfilled } else { reject(new Error("失敗")); // rejected } }, 1000); });

.then() / .catch() / .finally()

promise .then(result => { console.log("成功:", result); return result + " + 追加処理"; // 次の.thenへ }) .then(result2 => { console.log("次の処理:", result2); }) .catch(err => { console.log("エラー:", err.message); }) .finally(() => { console.log("成功/失敗どちらでも実行"); });
コールバック地獄との比較
Promise を使うと非同期処理を縦に並べて書けます。 ネストが深くならず、エラーハンドリングも1か所で済みます。

Promise.all() — 複数を並列実行

// 3つのAPIを同時に呼び出し、全部終わるのを待つ Promise.all([ fetchUser(), fetchOrders(), fetchProducts() ]).then(([user, orders, products]) => { // 全部揃ったら処理 console.log(user, orders, products); }).catch(err => { // どれか1つでも失敗したらここへ });

Promise.race() — 最初の完了を待つ

// タイムアウト処理などに使う Promise.race([ fetchData(), new Promise((_, reject) => setTimeout(() => reject(new Error("タイムアウト")), 5000) ) ]).then(result => console.log(result)) .catch(err => console.log(err.message));
