同期 vs 非同期

🏦

同期処理

銀行の窓口。
1人ずつ順番に対応。前の人が終わるまで次の人は待つ。

🍽️

非同期処理

レストランの注文。
複数テーブルに注文を取り、料理ができたテーブルから順番に提供。

同期処理 — 前の処理が終わるまで待つ

// 同期処理：上から順に実行される console.log("1: 開始"); console.log("2: 処理中"); console.log("3: 完了"); // 出力：1 → 2 → 3（必ずこの順）

JavaScriptはシングルスレッド

JavaScriptは1本のスレッドで動きます。重い処理をすると、その間ブラウザ全体が止まります。

ブロッキングの例（やってはいけない）
// 重い計算（同期）= ブロッキング let sum = 0; for (let i = 0; i < 1_000_000_000; i++) { sum += i; } // この間、ボタンのクリックも画面更新も止まる！

非同期処理 — 待たずに次へ進む

// setTimeout で非同期処理を体験 console.log("1: 開始"); setTimeout(function() { console.log("3: タイマー完了"); // 2秒後に実行 }, 2000); console.log("2: すぐ次に進む"); // 出力：1 → 2 → 3（2と3が逆！）
ポイント
非同期処理は「待ち時間」の間に別の処理を進められます。 setTimeoutの中身は2秒後に実行されますが、その間「2」は先に出力されます。

WEBで非同期が重要な理由

APIへのデータ取得（fetch）・DBへのアクセス・ファイル読み込みなど、応答待ちが発生する処理は全て非同期で書きます。

同期で書いてしまうと→画面がフリーズ→ユーザーが「壊れた」と思って離脱します。
