fetch API とは？

ブラウザからサーバーへHTTPリクエストを送るためのAPIです。
フロントエンドがバックエンドAPIと通信する時に使います。

HTTPメソッド

メソッド用途例
GETデータを取得一覧・詳細表示
POSTデータを送信・作成新規登録
PUTデータを更新（全体）編集・更新
DELETEデータを削除削除

GETリクエスト（基本形）

// async/await を使った fetch async function getUsers() { const response = await fetch('/api/users'); // HTTPステータスのチェック if (!response.ok) { throw new Error(`HTTP error: ${response.status}`); } const data = await response.json(); // JSONパース return data; }
fetchの落とし穴：HTTPエラーでもthrowしない
fetch は 404/500 でも reject されません。
必ず response.ok または response.status を確認してください。

POSTリクエスト（データ送信）

async function createOrder(orderData) { const response = await fetch('/api/orders', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(orderData) // オブジェクト→JSON文字列 }); if (!response.ok) { throw new Error(`エラー: ${response.status}`); } return response.json(); }

フロントエンド・バックエンド間の通信フロー

実際の流れ
① フロント: fetch('/api/users') を呼ぶ
② バックエンド（C#等）: SQLを実行してデータを取得
③ バックエンド: JSON形式でレスポンス
④ フロント: response.json() でパース → 画面に表示

フロントはAPIのURLを知るだけでよく、SQLやDBの詳細を知らなくてよい

JSONPlaceholder（練習用API）

// 無料で使える練習用REST API // https://jsonplaceholder.typicode.com/ GET /users → ユーザー一覧 GET /posts → 投稿一覧 GET /users/1 → id=1のユーザー POST /posts → 投稿作成（疑似）
