ORDER BY: 並び替え

取得したデータを特定の列で並び替えます。

基本構文

SELECT 列名 FROM テーブル名 ORDER BY 列名;

昇順（ASC）と降順（DESC）

-- 価格の安い順（昇順・デフォルト） SELECT * FROM products ORDER BY price ASC; -- 価格の高い順（降順） SELECT * FROM products ORDER BY price DESC;
ポイント
・ASC（Ascending）: 昇順（小→大）、省略可
・DESC（Descending）: 降順（大→小）

複数列での並び替え

-- カテゴリ順、同じカテゴリ内は価格順 SELECT * FROM products ORDER BY category_id ASC, price DESC;

LIMIT: 件数制限

取得する件数を制限します。

-- 上位3件だけ取得 SELECT * FROM products LIMIT 3; -- 価格が高い順で上位3件 SELECT * FROM products ORDER BY price DESC LIMIT 3;

OFFSET: スキップ

-- 3件目から5件取得（ページング用） SELECT * FROM products ORDER BY id LIMIT 5 OFFSET 2;
SQL Serverでは TOP を使う
本教材はSQLite（LIMIT）ですが、SQL Serverでは
SELECT TOP 3 * FROM products
のように書きます。

WHERE + ORDER BY + LIMIT

-- 食品カテゴリの中で価格が高い上位2件 SELECT * FROM products WHERE category_id = 1 ORDER BY price DESC LIMIT 2;
products: id, name, category_id, price, stock

練習問題

問1: 価格の昇順

全商品を価格の安い順に取得してください。


問2: 在庫の降順

全商品を在庫数の多い順に取得してください。


問3: 上位3件

価格が高い順に上位3件を取得してください。


問4: 複数列でソート

category_id の昇順で並べ、同じカテゴリ内では在庫数の降順で並べてください。


問5: WHERE + ORDER BY

飲料カテゴリ（category_id = 2）の商品を、価格の安い順で取得してください。


問6: 在庫が少ない商品

在庫数が少ない順に上位3件を取得してください。
これで「発注が必要な商品」が分かります。


問7: ページング

id順で4番目から3件取得してください。（2ページ目のイメージ）
ヒント: OFFSET 3 で最初の3件をスキップ
