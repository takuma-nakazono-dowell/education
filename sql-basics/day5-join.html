Day 5: テーブル結合（JOIN）

JOINとは？

複数のテーブルを関連する列で結合し、一緒に取得します。

products (商品)
├ id: 1, name: りんご, category_id: 1

categories (カテゴリ)
└ id: 1, name: 食品

→ JOINで結合すると
  りんご | 食品

INNER JOIN

両方のテーブルに一致するデータのみ取得します。

SELECT products.name AS 商品名, categories.name AS カテゴリ名 FROM products INNER JOIN categories ON products.category_id = categories.id;
ONの後ろで結合条件を指定
products.category_id = categories.id
「商品のcategory_idとカテゴリのidが一致するもの」

テーブルに別名をつける

-- p, c という短い別名を使う SELECT p.name AS 商品名, c.name AS カテゴリ名, p.price FROM products p INNER JOIN categories c ON p.category_id = c.id;

LEFT JOIN

左側のテーブルは全て取得し、右側は一致するものだけ。

-- 全商品を取得（カテゴリがなくてもOK） SELECT p.name, c.name AS category FROM products p LEFT JOIN categories c ON p.category_id = c.id;

3つ以上のテーブルを結合

-- orders + products + categories SELECT o.order_date, p.name AS 商品名, c.name AS カテゴリ, o.quantity FROM orders o INNER JOIN products p ON o.product_id = p.id INNER JOIN categories c ON p.category_id = c.id;

JOINと集計の組み合わせ

-- カテゴリ名ごとの商品数 SELECT c.name AS カテゴリ, COUNT(*) AS 商品数 FROM products p INNER JOIN categories c ON p.category_id = c.id GROUP BY c.name;
products: id, name, category_id, price, stock
categories: id, name
orders: id, product_id, quantity, order_date

練習問題

問1: 基本的なJOIN

products と categories を結合して、商品名とカテゴリ名を取得してください。


問2: JOINして全情報取得

商品名、カテゴリ名、価格、在庫を取得してください。


問3: JOIN + WHERE

「食品」カテゴリの商品だけを取得してください。


問4: カテゴリ別の集計

カテゴリ名ごとの商品数と平均価格を取得してください。


問5: 注文データと商品を結合

orders と products を結合して、注文日、商品名、数量を取得してください。


問6: 3テーブル結合

orders、products、categories を結合して
注文日、商品名、カテゴリ名、数量を取得してください。


問7: 商品ごとの注文集計

商品名ごとの合計注文数量を取得してください。
注文数量が多い順に並べてください。


問8: カテゴリ別の売上集計

カテゴリ名ごとに、注文された商品の合計数量と売上金額（price × quantity の合計）を取得してください。
