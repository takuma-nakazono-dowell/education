変数とは？

変数は「データを入れておく箱」のようなものです。名前をつけて、後から使うことができます。

変数の宣言: let と const

JavaScriptでは let または const を使って変数を宣言します。

let name = "田中"; // 後から変更できる const age = 25; // 後から変更できない
ポイント: let と const の使い分け
・let: 後から値を変更する可能性がある場合
・const: 値を変更しない場合（基本はこちらを使う）

データ型

変数に入れられるデータには種類（型）があります。

// 文字列（String）: ダブルクォートかシングルクォートで囲む const greeting = "こんにちは"; const city = '東京'; // 数値（Number）: クォート不要 const price = 1500; const tax = 0.1; // 真偽値（Boolean）: true または false const isActive = true; const isDeleted = false; // undefined: 値が設定されていない let something; // undefined になる // null: 意図的に「空」を表す const empty = null;

console.log() で確認する

変数の中身を確認するには console.log() を使います。

const name = "山田"; console.log(name); // 山田 と表示される const age = 30; console.log("年齢は", age, "歳です"); // 年齢は 30 歳です と表示される

文字列の連結

文字列同士は + で繋げられます。テンプレートリテラル（バッククォート）も便利です。

const firstName = "太郎"; const lastName = "山田"; // + で連結 const fullName1 = lastName + " " + firstName; // テンプレートリテラル（おすすめ） const fullName2 = `${lastName} ${firstName}`; console.log(fullName2); // 山田 太郎
よくあるミス
・文字列のクォートを閉じ忘れる
・const で宣言した変数に再代入しようとする
・変数名のスペルミス

変数の命名ルール

  • 英字、数字、_（アンダースコア）が使える
  • 数字から始めてはいけない
  • キャメルケース（firstName, lastName）が一般的
  • 意味のある名前をつける
