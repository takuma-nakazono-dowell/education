インターフェースと抽象クラス

インターフェースとは

「できること」を定義する契約。実装は持たない（メソッドのシグネチャのみ）。

// インターフェースの定義 interface ISaveable { void Save(); bool Load(string path); } interface IPrintable { void Print(); } // 複数インターフェースを実装できる（多重継承の代替） class Document : ISaveable, IPrintable { public string Content { get; set; } public void Save() { Console.WriteLine($"保存しました: {Content}"); } public bool Load(string path) { Console.WriteLine($"読み込み: {path}"); return true; } public void Print() { Console.WriteLine($"印刷: {Content}"); } }
インターフェースの役割: インターフェースは「何ができるか」を定義する。クラスは1つしか継承できないが、インターフェースは複数実装できる。

抽象クラス（abstract class）

// 抽象クラス: 直接インスタンス化できない abstract class Shape { public string Color { get; set; } // 抽象メソッド: 必ず派生クラスで実装する public abstract double Area(); // 通常のメソッドも持てる（インターフェースとの違い） public void Describe() { Console.WriteLine($"色: {Color}, 面積: {Area():F2}"); } } class Circle : Shape { public double Radius { get; } public Circle(double r) { Radius = r; } public override double Area() => Math.PI * Radius * Radius; } // new Shape() はエラー: 抽象クラスは直接生成不可

インターフェース vs 抽象クラス

インターフェース 抽象クラス
実装の共有 ✗（default実装は可） ✓
多重継承 ✓（複数実装可） ✗（1つのみ）
フィールド ✗ ✓
用途 「できること」の契約 共通基盤の提供
C# 8.0 以降: インターフェースに default メソッド実装が書けるようになった。ただし基本は実装なしが原則。

よく使う組み込みインターフェース

// IComparable: 比較可能にする class Temperature : IComparable<Temperature> { public double Value { get; } public Temperature(double v) { Value = v; } public int CompareTo(Temperature? other) { return Value.CompareTo(other?.Value ?? 0); } } // IEnumerable: foreachで反復可能にする（高度） // IDisposable: usingブロックで自動解放

練習問題

問題 1 easy

以下のコードを読んで、出力結果を答えてください。

interface IGreet { string Greet(); } class Japanese : IGreet { public string Greet() => "こんにちは"; } class English : IGreet { public string Greet() => "Hello"; } IGreet[] greeters = { new Japanese(), new English() }; foreach (var g in greeters) Console.WriteLine(g.Greet());
答えを見る
こんにちは Hello // 解説: インターフェース型の配列でも、 // 実際のクラスのメソッドが呼ばれる（ポリモーフィズム）

interface IArea { double GetArea(); } class Square : ____ { public double Side { get; } public Square(double side) { Side = side; } public ____ double GetArea() => Side * Side; }

問題 3 medium

IResizable インターフェースを定義し、実装してください。

  • メソッド: void Resize(double factor) / double GetSize()
  • Circle と Rectangle でそれぞれ実装する
  • Circleの「サイズ」は半径、Rectangleの「サイズ」は面積とする
答えを見る
interface IResizable { void Resize(double factor); double GetSize(); } class Circle : IResizable { private double _radius; public Circle(double radius) { _radius = radius; } // factorを掛けて半径を拡大・縮小 public void Resize(double factor) => _radius *= factor; // サイズ = 半径 public double GetSize() => _radius; } class Rectangle : IResizable { private double _width; private double _height; public Rectangle(double width, double height) { _width = width; _height = height; } // 幅・高さ両方にfactorを掛ける public void Resize(double factor) { _width *= factor; _height *= factor; } // サイズ = 面積 public double GetSize() => _width * _height; } // 使用例 IResizable circle = new Circle(5); Console.WriteLine($"変更前: {circle.GetSize()}"); // 5 circle.Resize(2); Console.WriteLine($"変更後: {circle.GetSize()}"); // 10 IResizable rect = new Rectangle(4, 3); Console.WriteLine($"変更前: {rect.GetSize()}"); // 12 rect.Resize(0.5); Console.WriteLine($"変更後: {rect.GetSize()}"); // 3

問題 4 medium

抽象クラスを使って動物クラス階層を作ってください。

  • abstract class Animal を作り、abstract void MakeSound() と具体的なメソッド void Sleep() を持たせる
  • Dog・Cat で実装し、Animal[] に入れてforeachで MakeSound() を呼ぶ
答えを見る
abstract class Animal { public string Name { get; } public Animal(string name) { Name = name; } // 抽象メソッド: 派生クラスで必ず実装する public abstract void MakeSound(); // 通常のメソッド: 共通の実装を持てる public void Sleep() { Console.WriteLine($"{Name}が眠った...zzz"); } } class Dog : Animal { public Dog(string name) : base(name) { } public override void MakeSound() { Console.WriteLine($"{Name}: ワン！"); } } class Cat : Animal { public Cat(string name) : base(name) { } public override void MakeSound() { Console.WriteLine($"{Name}: ニャー！"); } } // 使用例 Animal[] animals = { new Dog("ポチ"), new Cat("タマ"), new Dog("ハチ") }; foreach (Animal a in animals) { a.MakeSound(); } // 出力: // ポチ: ワン！ // タマ: ニャー！ // ハチ: ワン！

問題 5 hard

設計の選択について考えてください。

以下の要件を満たすには、インターフェースと抽象クラスのどちらが適切か説明し、コードを書いてください。

  • 「支払い」という概念があり、クレジットカード・電子マネー・現金の3種類がある
  • 全て Pay(decimal amount) を持つ
  • クレジットカードと電子マネーは共通の CheckBalance() 残高確認ロジックを持つ
答えを見る
// 設計の考え方: // - 全クラスが共通の Pay() を持つ → インターフェース IPayable で契約 // - クレジットカード・電子マネーは共通ロジックを持つ → 抽象クラス BalancePayment // - 現金は残高管理なし → IPayable を直接実装 // → インターフェース + 抽象クラスの組み合わせが最適 // 全支払い方法の契約 interface IPayable { void Pay(decimal amount); } // 残高管理が必要な支払い方法の共通基底 abstract class BalancePayment : IPayable { protected decimal _balance; public BalancePayment(decimal initialBalance) { _balance = initialBalance; } // 共通の残高確認ロジック public void CheckBalance() { Console.WriteLine($"残高: {_balance:C}"); } // Pay()は各クラスで実装 public abstract void Pay(decimal amount); } // クレジットカード（後払いなので残高＝与信枠） class CreditCard : BalancePayment { public CreditCard(decimal creditLimit) : base(creditLimit) { } public override void Pay(decimal amount) { if (amount <= _balance) { _balance -= amount; Console.WriteLine($"クレジット支払い: {amount:C}"); } else { Console.WriteLine("与信枠不足"); } } } // 電子マネー（チャージ残高） class ElectronicMoney : BalancePayment { public ElectronicMoney(decimal balance) : base(balance) { } public override void Pay(decimal amount) { if (amount <= _balance) { _balance -= amount; Console.WriteLine($"電子マネー支払い: {amount:C}"); } else { Console.WriteLine("残高不足"); } } } // 現金（残高管理不要） class Cash : IPayable { public void Pay(decimal amount) { Console.WriteLine($"現金支払い: {amount:C}"); } } // ポリモーフィズムで扱える IPayable[] payments = { new CreditCard(100000), new ElectronicMoney(5000), new Cash() }; foreach (IPayable p in payments) { p.Pay(3000); }
