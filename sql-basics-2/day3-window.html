ウィンドウ関数とは？

GROUP BY のように集計しながら、行を消さずに結果を出せる関数です。
「ランキング」「累計」「グループ内の順位」などが簡単に書けます。

GROUP BY との違い
GROUP BY → 行をまとめて件数が減る
ウィンドウ関数 → 全行を保ちながら集計値を追加

基本構文

関数名() OVER ( [PARTITION BY 列名] -- グループを分ける [ORDER BY 列名] -- 並び順 )

① ROW_NUMBER() — 連番

-- カテゴリ内で価格が高い順に連番を付ける SELECT name, category_id, price, ROW_NUMBER() OVER ( PARTITION BY category_id ORDER BY price DESC ) AS row_num FROM products;

② RANK() / DENSE_RANK() — 順位

同じ値のとき順位が変わります。

SELECT name, price, RANK() OVER (ORDER BY price DESC) AS rnk, DENSE_RANK() OVER (ORDER BY price DESC) AS d_rnk FROM products;

③ SUM() OVER — 累計・グループ合計

-- カテゴリ内の合計在庫数を各行に付加 SELECT name, category_id, stock, SUM(stock) OVER ( PARTITION BY category_id ) AS category_stock_total FROM products;

使用するテーブル

products: id, name, category_id, price, stock
order_details: id, order_id, product_id, quantity

練習問題

問1: ROW_NUMBER — 全体ランキング

全商品を価格の高い順に並べ、ROW_NUMBERで連番（row_num）を付けてください。
列名：row_num, name, price


問2: RANK — カテゴリ内価格順位

カテゴリ内での価格順位（高い順）をRANKで付けてください。
列名：category_id, name, price, price_rank


問3: SUM OVER — カテゴリ別合計在庫

各商品にカテゴリ全体の合計在庫数（category_stock）を付けて表示してください。
列名：category_id, name, stock, category_stock


問4: ウィンドウ関数 + WHERE — カテゴリ内1位の商品

カテゴリ内で価格が最も高い商品（RANK = 1）だけを取得してください。
ヒント：FROM サブクエリでウィンドウ関数を計算してから WHERE で絞り込みます。
