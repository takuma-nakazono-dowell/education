相関サブクエリとは？

Day 1のサブクエリは「内側を先に実行して結果を渡す」方式でした。
相関サブクエリは、外側クエリの「1行ずつ」に対してサブクエリを実行します。

イメージ

-- 各商品より高い商品が何件あるかを数える SELECT p1.name, p1.price, (SELECT COUNT(*) FROM products p2 WHERE p2.price > p1.price) AS higher_count FROM products p1;
ポイント: p1 が外側クエリの行を指す
サブクエリ内で p1.price を参照することで、外側の1行ごとに結果が変わります。

① EXISTS / NOT EXISTS

「サブクエリが1件以上ヒットするか？」を判定します。実際の値は使いません。

-- 注文があった顧客だけを取得 SELECT c.name, c.region FROM customers c WHERE EXISTS ( SELECT 1 FROM orders o WHERE o.customer_id = c.id );
-- 一度も注文していない顧客 SELECT c.name, c.region FROM customers c WHERE NOT EXISTS ( SELECT 1 FROM orders o WHERE o.customer_id = c.id );

② IN vs EXISTS — どちらを使うべきか？

WHERE IN

  • シンプルで読みやすい
  • サブクエリの結果件数が少ないとき有利
  • NULLが含まれると意図しない動作になることがある

EXISTS

  • 1件見つかった時点で探索を止める
  • サブクエリの結果が大量でも速い
  • NULLの影響を受けない
実務の判断基準
「存在するかどうか」だけを判定するなら EXISTS を選ぶと安全でパフォーマンスも安定します。 値を使いたい場合は IN または JOIN を検討します。

使用するテーブル

customers: id, name, region
orders: id, customer_id, order_date
order_details: id, order_id, product_id, quantity
products: id, name, category_id, price, stock
データベースを準備中...

練習問題

問1: EXISTS — 注文がある顧客

注文（orders）が1件以上ある顧客の name と region を取得してください。（EXISTS を使用）


問2: NOT EXISTS — 注文がない顧客

一度も注文していない顧客の name と region を取得してください。


問3: 相関サブクエリ — 同一カテゴリ内での順位付け

各商品について、同じカテゴリ内で自分より高い価格の商品が何件あるかを求めてください。
列名：name, category_id, price, higher_in_category
（0件 = そのカテゴリで最も高い商品）


問4: IN vs EXISTS — 動作確認

order_detailsに登録されている product_id の商品を取得してください。
まず IN で試し、次に EXISTS で同じ結果になることを確認しましょう。
