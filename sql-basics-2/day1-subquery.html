サブクエリとは？

SELECT文の中に書く、もう一つのSELECT文です。「クエリの中のクエリ」なのでサブクエリ（副問い合わせ）と呼びます。

① WHERE IN サブクエリ

「Xに含まれる」という条件を動的に作れます。

-- 注文されたことがある商品だけを取得 SELECT name, price FROM products WHERE id IN ( SELECT product_id FROM order_details );
ポイント
内側のSELECT（サブクエリ）が先に実行され、その結果リストを外側のWHERE INが使います。

② スカラーサブクエリ

1行1列の値を返すサブクエリです。SELECT句やWHERE句で値として使えます。

-- 平均価格より高い商品を取得 SELECT name, price FROM products WHERE price > ( SELECT AVG(price) FROM products );
-- 各商品の価格と平均価格を並べて表示 SELECT name, price, (SELECT AVG(price) FROM products) AS avg_price FROM products;

③ FROM サブクエリ（インラインビュー）

サブクエリをテーブルのように扱います。集計結果に対してさらに絞り込みたいときに使います。

-- カテゴリ別の平均価格を求め、 -- その中で平均価格が150以上のものを取得 SELECT * FROM ( SELECT category_id, AVG(price) AS avg_price FROM products GROUP BY category_id ) AS t WHERE t.avg_price >= 150;
注意: サブクエリのパフォーマンス
サブクエリは便利ですが、データが多い場合は遅くなることがあります。 特に「FROM サブクエリ」で大量データを集計する場合は注意が必要です（Day 5で詳しく学びます）。

使用するテーブル

products: id, name, category_id, price, stock
order_details: id, order_id, product_id, quantity
データベースを準備中...

練習問題

問1: WHERE IN サブクエリ

注文されたことがある商品（order_detailsにproduct_idが存在する商品）の name と price を取得してください。


問2: WHERE NOT IN サブクエリ

一度も注文されていない商品の name と price を取得してください。（NOT IN を使用）


問3: スカラーサブクエリ（WHERE句）

全商品の平均価格より高い商品の name と price を取得してください。


問4: スカラーサブクエリ（SELECT句）

全商品の name・price・全体平均価格（avg_price）を並べて取得してください。
全体平均価格はスカラーサブクエリで取得します。


問5: FROM サブクエリ

カテゴリ別の平均価格を求め、その中で平均価格が 140 以上のカテゴリを取得してください。
列名：category_id, avg_price
